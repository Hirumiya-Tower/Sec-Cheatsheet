# 解説
コマンドラインベースのパケットキャプチャツール。
Wireshark用のpcapを生成することも、pcapを読み込むこともできる。

# 基本構文

```tcpdump [オプション] [フィルタ]```

# 有用なオプション

### 基本操作

```-i <interface>``` 特定のインターフェイス（eth0, anyなど）をキャプチャ
```-D``` 使用可能インターフェイスを一覧表示。
```-c <count>``` 指定した行数のパケットを受信して終了。
```-n``` IPアドレスをホスト名に解決しない。
```-n``` IPアドレスもポート番号も解決しない。
```-w <file>.pcap``` 結果を表示せず、指定ファイルに保存する。
```-r <file>.pcap``` 保存されたpcapファイルを読み込んで表示する。

# 表示・出力制御

```-v, -vv, -vvv``` 詳細表示
```-X``` パケット内容を16進数（Hex）とASCIIの両方で表示
```-A``` パケット内容をASCIIで表示

# フィルタ式

### 基本ターゲット

```host <IP>``` 特定のIPに関する通信を表示。
```src <IP> / dst <IP>``` 送信元、送信先のIPを指定。
```port <PORT>``` 特定のポート番号を指定。
```net <Network/Mask>``` 特定のサブネット（```192.168.1.0/24``` など）を指定

### プロトコル指定

```tcp, udp, icmp, arp``` 特定のプロトコルに限定。

### 論理演算

```and, or, not``` 等の組み合わせが可能

# コマンド例

- **とにかくASCIIで中身を読みたい時**
	```tcpdump -i eth0 -nn -A``` 
- **特定のIPとの通信をpcapに保存したい時**
	```tcpdump -i any -nn host 10.10.10.10 -w export.pcap	```
- **特定のポートへのSYNパケットだけを見たい時**
	```tcpdump -i eth0 'port 80 and tcp[tcpflags] & tcp-syn != 0'	```
	ビット演算子'&'などを使うときは、必ずシングルクォーテーションで囲う。
